RECALL is a declarative web interface language with COBOL-inspired syntax. It transpiles to self-contained, dependency-free HTML files. The source is always embedded in the output.

WHY COBOL

Why COBOL.

AI'S BEST TRAINING LANGUAGE.

COBOL has been encoding business intent in plain English since 1959. More semantic signal per line than any modern language. The intent is readable, not inferred. That is not a coincidence — it is the design.

SINCE 1959

SEMANTIC INTENT BY CONSTRUCTION.

Every .rcl file is a semantic intent document. An AI can read, modify, generate, and audit RECALL source without a schema — because the intent is in the syntax, not hidden behind it.

BY DESIGN

BUILT FOR THE AI COLLABORATION LOOP.

HTML has no grammar. An AI generating HTML can do anything — including things you did not ask for. RECALL's constrained syntax makes AI output predictable, round-trippable, and auditable. Ask for a page. Get back valid .rcl. Compile. Done.

DESIGN PRINCIPLES

What RECALL believes.

THE SOURCE IS THE ARTIFACT

Every compiled .html file embeds its .rcl source in a comment block at the top. View source on any RECALL page and you will see COBOL divisions.

CORE

ZERO DEPENDENCIES

Output is one HTML file. Inline CSS. Vanilla JS where needed. No CDN links. No frameworks. No build step to run the result.

CONSTRAINT

VERBOSE IS A FEATURE

COBOL's English-like verbosity is intentional legibility. A RECALL source file reads like a structured document. The intent is visible without a comment.

PHILOSOPHY

LEGACY FORWARD

This is not nostalgia. COBOL systems from the 1960s are still running payroll. RECALL asks: what would a web language look like if it was designed to last?
